Section 6.1|Introduction

\(\S\) 6.1.1 - Purposes of Probability Plots

STAT 687 - Chapter 6

In practical applications probability plots are used to:

  • Assess the adequacy of a particular distributional model

  • Provide nonparametric graphical estimates of probabilities and distribution quantiles

  • Obtain graphical estimates of parametric model parameters (e.g.by fitting a straight line through the points on a probability plot)

  • Display the results of a parametric maximum likelihood fit along with the data

In addition, probability plots often reveal information about a population, a process, or data that might otherwise escape detection

\(\S\) 6.1.2 - Prob Plotting Scales: Linearizing a cdf

STAT 687 - Chapter 6

CDFs from different distributions can appear similar on nonparametric plots

Probability plots use transformed axes to plot a linearized cdf vs \(t\)

Basic Idea

  • Invert \(F(t_{p})\) to analytically determine the quantile function \(t_{p}\)

  • Transform \(F^{-1}(p)\) to obtain a linear plot of the form

\[ \begin{aligned} y&=m(x)\;+b\\\\ F^{-1}(p)&=m(t_{p})+b \end{aligned} \]

Assessing the adequacy of a distributions fit for a data set is an important application of probability plots

Probability plots are created by

  1. Identifying the points \(\left(t_{i},\hat{F}(t_{i})=\hat{p}_{i}\right), \;\;\;\;i=1,2,...,n\)

  3. Observe if the data fall approximately on a straight line

Section 6.4|Probability Plotting Postions

\(\S\) 6.4.1 - Criteria for Choosing Plotting Positions

STAT 687 - Chapter 6

Probability plots require a way to compute plotting positions \(F(t_{i}), i=1,...,n\)

Choosing a plotting position formula depends on the purpose of the plot

  • Checking distributional assumptions
  • Estimating distribution parameters

The best plotting position formula is a function of

  • The assumed underlying model
  • The function to be estimated \((t_{p}, h(t), F(t),...)\)
  • The type and amount of censoring

Most texts oversimplify the choice of a plotting position formula

\(\S\) 6.4.1 - Criteria for Choosing Plotting Positions

STAT 687 - Chapter 6

The generic plotting position formula is

\[p_{i}=\frac{i-a}{n+1-2a}\]

where

\[ \begin{aligned} i &\equiv\;\text{an index of the ordered observations (smallest}\rightarrow\text{largest)}\\ n &\equiv\;\text{the sample size}\\ a &\equiv\;\text{the plotting position parameter}\\ p_{i}&\equiv\;\text{the value of the plotting position corresponding to time}\; t_{i} \end{aligned} \]

A value of \(a\) is chosen to produce approximately unbiased estimates for \(F(t_{i})\) for an assumed distribution

\(\S\) 6.4.1 - Criteria for Choosing Plotting Positions

STAT 687 - Chapter 6

Except for specialized situations, the plotting positions used in practice are

\[ \begin{aligned} \text{Hazen}\; &p_{i}=\frac{i-0.5}{n}\\\\ \text{Chegodayev}\; &p_{i}=\frac{i-0.3}{n+0.4}\;\text{also known as "Median-Ranks"}\\\\ \text{Weibull}\; &p_{i}=\frac{i}{n+1}\\\\ \end{aligned} \]

Plotting positions for this text

\[ p_{i}=\begin{cases}\frac{i}{n}, &\mbox{Distribution free - (Eq 3.1)}\\\\ \frac{i-0.5}{n}, &\mbox{Assumed distribution}\end{cases} \]

\(\S\) 6.4.2 - Choice of Plotting Positions

STAT 687 - Chapter 6

Censoring considerations for selecting a plotting position formula

  1. Continuous inspection or small inspection intervals
  2. Interval censoring with large intervals
  3. Arbitrary censoring
    • Left censoring
    • Right censoring
    • Interval censoring
    • Overlapping intervals

\(\S\) 6.4.2 - Choice of Plotting Positions

STAT 687 - Chapter 6

Case 1 - Data with continuous inspection and a single censoring event

A logical estimator is

\[\hat{p}_{i}=\frac{i}{n}\]

Estimator properties

  • Increases by \(\frac{1}{n}\) at each failure time \(t_{i}\)
  • Underestimates \(F(t)\) prior to the first failure time at \(t_{1}\)
  • Overestimates \(F(t)\) after the last failure time at \(t_{n}\)

The meekR package uses this estimator for plot(data.ld)

A compromise plotting position for addressing this bias uses midpoint of the \(1/n\) jumps

\[\hat{p}_{i}=\frac{i-0.5}{n}=\frac{1}{2}\left[\hat{F}(t_{(i)})+\hat{F}(t_{(i-1)})\right]\]

The meekR package uses this comprimise estimator for plot(data.ld, distribution = "distribution")

\(\S\) 6.4.2 - Choice of Plotting Positions

STAT 687 - Chapter 6

Case 2 - Data with continuous inspection and multiple right censoring events

For this case, each failure can result in a jump that is larger that \(1/n\)

  • Censored observations between failures will reduce \(n\)

  • Furthermore, \(\frac{i-0.5}{n}\ne\frac{1}{2}\left[\hat{F}(t_{(i)})+\hat{F}(t_{(i-1)})\right]\)

For this case, we use the estimated values \(\hat{F}(t_{(i)})\) directly as

\[\hat{p_{i}}=\frac{1}{2}\left[\hat{F}(t_{(i)})+\hat{F}(t_{(i-1)})\right]\]

where \(\hat{F}(t_{(i)})\;\text{and}\;\hat{F}(t_{(i-1)})\) are computed using Eqs. 3.6 & 3.7.

\(\S\) 6.4.2 - Choice of Plotting Positions

STAT 687 - Chapter 6

Recall from Eq. 3.7

\[\hat{F}(t_{i})=1-\prod_{j=1}^i \left[1-\hat{p}_{j}\right]=1-\prod_{j=1}^i \left[1-\frac{d_j}{n_j}\right]\]

Therefore,

\[ \begin{aligned} p_i=\frac{1}{2}\left[1-\prod_{j=1}^i \left[1-\frac{d_j}{n_j}\right]+1-\prod_{j=1}^{i-1} \left[1-\frac{d_j}{n_j}\right]\right] \end{aligned} \]

Case 3 - Data with Interval censoring

  • Failures occur at the end of an interval when discovered by inspection
  • \(\hat{F}(t)\) defined as the upper endpoint of an interval in which failures occurred
  • For this case, the plotting position is

\[p_{i}=\hat{F}(t_{(i)})\]

Where

  • \(E\left[\hat{F}(t_{(i)})\right]=F(t_{(i)})\) when there are no losses due censoring
  • \(E\left[\hat{F}(t_{(i)})\right]\approx F(t_{(i)})\) when multiple censoring does occur

\(\S\) 6.6.2 - Reasons for bends in prob. plots

STAT 687 - Chapter 6

Sharp bends or departures from linearity could indicate the existence of

  • Multiple failure modes

  • Different usage patterns in the sample

  • An unexpected or unaccounted stressor

For this reason reliability data should include

  • The failure mode(s) observed

  • Variations in the usage environment

  • Variations in the amount of usage
